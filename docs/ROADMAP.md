# 项目路线图

本文件是项目计划与阶段状态的唯一维护入口。README 负责说明项目入口、架构和复现方式；`experiments/` 负责记录已经发生的实验；本文件负责描述下一步准备验证什么。

## 总体原则

- 先建立基线和评估能力，再做优化。
- 每次实验只改变一个关键变量。
- 策略对比固定评估集、模型配置和随机性控制。
- 指标变化必须结合典型案例解释，不能只看单个平均分。
- 未经评估验证的方案只能标记为计划，不写作项目能力。

## 全局架构

```text
数据层：Markdown / PDF -> 文档加载 -> Document
预处理层：切分 -> 元数据提取 -> 清洗
索引层：Embedding -> Chroma -> 可选 BM25
检索层：查询改写 -> 多路召回 -> 元数据过滤 -> Rerank
生成层：Prompt -> LLM -> 答案与来源
评估层：评估数据集 -> RAGAS -> 实验记录 -> A/B 报告
```

## 阶段路线

### Phase 0：环境与数据准备

目标：建立项目骨架，验证 Markdown 文档可以被加载和统计。

状态：已完成基础 Markdown 数据加载；PDF 加载与数据清点仍作为后续扩展。

验收：能够输出文档数量、长度与来源路径，无加载异常。

### Phase 1：Baseline RAG

目标：建立后续所有优化的对照组，并锁定基线参数。

当前基线：固定长度切分、局部 Embedding、Chroma 相似度检索、DeepSeek 上下文生成。

验收：能够回答基础问题，并保留可复现的配置、响应时间和典型案例。

### Phase 2：评估框架

目标：在任何优化开始前，先具备稳定打分能力。

当前进展：RAGAS 测试集生成与四项核心指标已跑通。

待完善：

- 扩展为 80-150 条混合评估集。
- 结合 RAGAS 合成问题、人工黄金问题和域外拒答问题。
- 验证同一版本重复运行时的指标波动，并保留评估集版本。

验收：能稳定复现分数，并能说明基线在何类问题上表现较差。

### Phase 3：检索优化

目标：逐个验证不同检索策略对召回、精度、忠实度和延迟的影响。

候选实验，不应默认全部实施：

1. 摘要检索：以摘要定位文档，再映射回原文。
2. 父子分块：小块检索、大块返回。
3. 句子窗口：句子级命中后扩展相邻上下文。
4. 混合检索：向量检索与 BM25 并行召回并融合。
5. Rerank：扩大候选集后使用重排模型选取最终上下文。

每个方向均使用同一评估集，并单独记录：配置、指标、延迟、成本与至少三个典型案例。

### Phase 4：生成优化

目标：在检索质量固定的前提下，提高答案忠实度与相关性。

候选实验：上下文约束 Prompt、答案与来源的结构化输出、幻觉抑制策略。

验收：Faithfulness 改善，并通过人工抽查验证幻觉案例减少。

### Phase 5：查询改写

目标：提升口语化、模糊问题的检索效果。

候选实验：查询扩展、多查询检索、HyDE。

验收：在单独标注的模糊问题子集上验证 Recall 变化，而不是只看全量平均分。

### Phase 6：元数据与多格式文档

目标：发挥 Markdown 结构信息，并接入 PDF。

计划内容：

- 从 frontmatter、标题层级和路径中提取元数据。
- 为 chunk 保存标题面包屑、文件路径、文档类型和位置索引。
- 实现元数据过滤与自然语言过滤查询。
- 为 PDF 保留页码和来源定位。

### Phase 7：组合优化与消融实验

目标：组合已经被独立实验验证有效的模块，不盲目堆叠功能。

验收：形成最终 Pipeline，并通过消融实验量化每个优化点的贡献、延迟和成本。

### Phase 8：开源发布准备

目标：让外部开发者可以在不依赖私人语料的情况下复现项目。

计划内容：

- 锁定 Python 与依赖版本，提供完整安装说明。
- 提供脱敏的小型 demo 数据集。
- 整理实验结果、案例和对比图表。
- 补充许可证、贡献指南与可选交互界面。

## 评估集策略

评估集应逐步由三类问题组成：

| 来源 | 建议比例 | 作用 |
| --- | ---: | --- |
| RAGAS 合成问题 | 50-60% | 覆盖一般知识点与不同难度 |
| 人工黄金问题 | 25-30% | 贴近真实使用场景，提供可靠参考答案 |
| 域外 / 对抗问题 | 10-15% | 评估拒答准确率与幻觉风险 |

域外问题应单独统计拒答准确率，不与普通知识问答的 RAGAS 平均分混合。

## 实验记录契约

每个实验应在 `experiments/` 留下独立记录，并在汇总表中保存：

```text
experiment_id
category
variant
dataset_version
context_precision
context_recall
faithfulness
answer_relevancy
avg_latency_ms
avg_token_cost
notes
```

推荐模板：

```markdown
### YYYY-MM-DD - 实验名称
- 目标：
- 假设：
- 唯一变量：
- 固定条件：
- 数据与评估集版本：
- 指标结果：
- 典型案例：
- 结论：
- 下一步：
```

## Markdown 场景优先事项

当进入切分和元数据优化阶段，优先验证：

- 标题层级切分后再对超长段落做二次切分。
- 代码块完整性保护。
- 为 chunk 拼接标题面包屑，避免局部文本丢失主题。
- 短笔记不做过度切分。
- 从 frontmatter 或文件系统回退提取标题、标签、日期和路径。
